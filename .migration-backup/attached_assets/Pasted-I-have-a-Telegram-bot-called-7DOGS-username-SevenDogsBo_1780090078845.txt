- fp.html generates a unique device fingerprint (based on browser/device data)
- This fingerprint is sent to the backend and saved with the user's Telegram ID

### What needs to be integrated:

   - Do NOT count the referral reward for duplicate accounts

4. **Block duplicate users** - if a user is detected as duplicate:
   - Show them a message: "This device is already registered with another account"
   - Do not give them coins, spins, or referral rewards

   show a "⚠️ Duplicate Device" badge next to flagged accounts

6. **The fingerprint check must happen:**
